===============================================================================
                      RECONCILIATION LOGIC
===============================================================================

ALGORITHM: SHA-256 Hash-Based Comparison

HASH GENERATION:
  Input: vendor_name | amount | billing_cycle | line_item_id
  Excluded: id (as specified)
  Null Handling: Null amounts represented as "NULL" string
  
PROCESS:
  1. Fetch all records from source database
  2. Fetch all records from target database
  3. Generate SHA-256 hashes for each record
  4. Build in-memory hash maps
  5. Compare sets:
     - Matched records: Present in both with identical hashes
     - Missing in target: In source but not in target
     - Missing in source: In target but not in source
  6. Calculate statistics:
     - Match percentage
     - Discrepancy count
     - Source-only count
     - Target-only count

DISCREPANCY TYPES:
  - MISSING_IN_TARGET: Record in source but not target
  - MISSING_IN_SOURCE: Record in target but not source
  - DATA_MISMATCH: (Reserved for future enhancements)
